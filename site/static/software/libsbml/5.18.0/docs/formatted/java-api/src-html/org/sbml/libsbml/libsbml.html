704 * returned by this method.  Callers can use the methods on {@link SBMLDocument} such
705 * as
706 * ,
707 * {@link SBMLDocument#getNumErrors()}
708 * and
709 <p>

713 <p>
714 * {@link SBMLDocument#getError(long)}

717 * category, and severity level of the problem, as well as a textual
718 * description of the problem.  The possible severity levels range from

1203 * Abstract Syntax Tree (AST).
1204 <p>
1205 * <p>
1206 * The text-string form of mathematical formulas produced by
1207 * <a href='libsbml.html#formulaToString(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToString(ASTNode)</code></a> and read by
1208 * <a href='libsbml.html#parseFormula(java.lang.String)'><code>libsbml.parseFormula(String)</code></a> use a simple C-inspired infix
1209 * notation taken from SBML Level&nbsp;1.  A formula in this text-string form
1210 * therefore can be handed to a program that understands SBML Level&nbsp;1
1211 * mathematical expressions, or used as part of a formula translation system.
1212 * The syntax is described in detail in the documentation for {@link ASTNode}.  The
1213 * following are illustrative examples of formulas expressed using this syntax:
1214 * <pre class='fragment'>
12150.10 * k4^2
1216</pre>
1217<pre class='fragment'>
1218(vm * s1)/(km + s1)
1219</pre>
1220 <p>
1221 * Note that this facility is provided as a convenience by libSBML&mdash;the
1222 * MathML standard does not actually define a 'string-form' equivalent to
1223 * MathML expression trees, so the choice of formula syntax is somewhat
1224 * arbitrary.  The approach taken by libSBML is to use the syntax defined by
1225 * SBML Level&nbsp;1 (which in fact used a text-string representation of
1226 * formulas and not MathML).  This formula syntax is based mostly on C
1227 * programming syntax, and may contain operators, function calls, symbols,
1228 * and white space characters.  The following table provides the precedence
1229 * rules for the different entities that may appear in formula strings.
1230 <p>
1231 * <table border="0" class="centered text-table width80 normal-font alt-row-colors" style="padding-bottom: 0.5em">
1232 <tr style="background: lightgray; font-size: 14px;">
1233     <th align="left">Token</th>
1234     <th align="left">Operation</th>
1235     <th align="left">Class</th>
1236     <th>Precedence</th>
1237     <th align="left">Associates</th>
1238 </tr>
1239<tr><td><em>name</em></td><td>symbol reference</td><td>operand</td><td align="center">6</td><td>n/a</td></tr>
1240<tr><td><code>(</code><em>expression</em><code>)</code></td><td>expression grouping</td><td>operand</td><td align="center">6</td><td>n/a</td></tr>

2934/**
2935 * Returns a copy of the default Level&nbsp;3 ('L3') formula parser settings.
2936 <p>
2937 * The data structure storing the settings allows callers to change the
2938 * following parsing behaviors:
2939 <p>
2940 * <p>
2941 * The text-string form of mathematical formulas read by the function
2942 * <a href='libsbml.html#parseL3Formula(java.lang.String)'><code>libsbml.parseL3Formula(String)</code></a> and written by the function
2943 * <a href='libsbml.html#formulaToL3String(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToL3String(ASTNode)</code></a> uses an expanded version of
2944 * the syntax read and written by <a href='libsbml.html#parseFormula(java.lang.String)'><code>libsbml.parseFormula(String)</code></a>
2945 * and <a href='libsbml.html#formulaToString(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToString(ASTNode)</code></a>, respectively.  The
2946 * latter two libSBML functions were originally developed to support
2947 * conversion between SBML Levels&nbsp;1 and&nbsp;2, and were focused on the
2948 * syntax of mathematical formulas used in SBML Level&nbsp;1.  With time, and
2949 * the use of MathML in SBML Levels&nbsp;2 and&nbsp;3, it became clear that
2950 * supporting Level&nbsp;2 and&nbsp;3's expanded mathematical syntax would be
2951 * useful for software developers.  To maintain backwards compatibility for
2952 * libSBML users, the original <a href='libsbml.html#formulaToString(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToString(ASTNode)</code></a>
2953 * and <a href='libsbml.html#parseFormula(java.lang.String)'><code>libsbml.parseFormula(String)</code></a> have been left untouched,
2954 * and instead, the new functionality is provided in the form of
2955 * <a href='libsbml.html#parseL3Formula(java.lang.String)'><code>libsbml.parseL3Formula(String)</code></a> and
2956 * <a href='libsbml.html#formulaToL3String(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToL3String(ASTNode)</code></a>.

2958 * The following lists the main differences in the formula syntax supported by
2959 * the Level 3 ('L3') versions of the formula parsers and formatters,
2960 * compared to what is supported by the Level&nbsp;1-oriented
2961 * <a href='libsbml.html#parseFormula(java.lang.String)'><code>libsbml.parseFormula(String)</code></a> and
2962 * <a href='libsbml.html#formulaToString(org.sbml.libsbml.ASTNode)'><code>libsbml.formulaToString(ASTNode)</code></a>:
2963 <p>
2964 * <ul>
2965 * <li> Units may be asociated with bare numbers, using the following syntax:
2966 * <div style='margin: 10px auto 10px 25px; display: block'>
2967 * <span class='code' style='background-color: #d0d0ee'>number</span>
2968 * <span class='code' style='background-color: #edd'>unit</span>
2969 * </div>
2970 * The <span class='code' style='background-color: #d0d0ee'>number</span>
2971 * may be in any form (an integer, real, or rational
2972 * number), and the
2973 * <span class='code' style='background-color: #edd'>unit</span>
2974 * must conform to the syntax of an SBML identifier (technically, the
2975 * type defined as <code>SId</code> in the SBML specifications).  The whitespace between
2976 * <span class='code' style='background-color: #d0d0ee'>number</span>
2977 * and <span class='code' style='background-color: #edd'>unit</span>
2978 * is optional.
2979 <p>
2980 * <li> The Boolean function symbols <code>&&</code> (<em>and</em>), <code>||</code> (<em>or</em>), <code>!</code> (<em>not</em>),

2983 * <li> The <em>modulo</em> operation is allowed as the symbol <code>@%</code> and will
2984 * produce a <code>&lt;piecewise&gt;</code> function in the corresponding
2985 * MathML output by default, or can produce the MathML function <code>rem</code>, 
2986 * depending on the {@link L3ParserSettings} object (see 
2987 * L3ParserSettings_setParseModuloL3v2() ).

2989 * <li> All inverse trigonometric functions may be defined in the infix either
2990 * using <code>arc</code> as a prefix or simply <code>a</code>; in other words, both <code>arccsc</code>

2993 * oriented parser implemented by <a href='libsbml.html#parseFormula(java.lang.String)'><code>libsbml.parseFormula(String)</code></a>

2997 * as a numerical division:
2998 * <pre style='display: block; margin-left: 25px'>
2999 * (<span class='code' style='background-color: #d0d0ee'>integer</span>/<span class='code' style='background-color: #d0d0ee'>integer</span>)</pre>
