583/**
584   * Unsets the value of the 'id' attribute of this {@link GraphicalObject}.
585   <p>
586   * <p>
587 * <p>
588 * The identifier given by an object's 'id' attribute value
589 * is used to identify the object within the SBML model definition.
590 * Other objects can refer to the component using this identifier.  The
591 * data type of 'id' is always <code>SId</code> or a type derived
592 * from that, such as <code>UnitSId</code>, depending on the object in 
593 * question.  All data types are defined as follows:
594 * <pre style='margin-left: 2em; border: none; font-weight: bold; color: black'>
595 *   letter .= 'a'..'z','A'..'Z'
596 *   digit  .= '0'..'9'
597 *   idChar .= letter | digit | '_'
598 *   SId    .= ( letter | '_' ) idChar*
599 * </pre>
600 * The characters <code>(</code> and <code>)</code> are used for grouping,
601 * the character <code>*</code> 'zero or more times', and the character
602 * <code>|</code> indicates logical 'or'.  The equality of SBML identifiers
603 * is determined by an exact character sequence match; i.e., comparisons must
604 * be performed in a case-sensitive manner.  This applies to all uses of
605 * <code>SId</code>, <code>SIdRef</code>, and derived types.
606 <p>
607 * Users need to be aware of some important API issues that are the result of
608 * the history of SBML and libSBML.  Prior to SBML Level&nbsp;3
609 * Version&nbsp;2, SBML defined 'id' and 'name' attributes on only a subset
610 * of SBML objects.  To simplify the work of programmers, libSBML's API
611 * provided get, set, check, and unset on the {@link SBase} object class itself
612 * instead of on individual subobject classes. This made the
613 * get/set/etc. methods uniformly available on all objects in the libSBML
614 * API.  LibSBML simply returned empty strings or otherwise did not act when
615 * the methods were applied to SBML objects that were not defined by the SBML
616 * specification to have 'id' or 'name' attributes.  Additional complications
617 * arose with the rule and assignment objects: {@link InitialAssignment},
618 * {@link EventAssignment}, {@link AssignmentRule}, and {@link RateRule}.  In early versions of SBML,
619 * the rule object hierarchy was different, and in addition, then as now,
620 * they possess different attributes: 'variable' (for the rules and event
621 * assignments), 'symbol' (for initial assignments), or neither (for
622 * algebraic rules).  Prior to SBML Level&nbsp;3 Version&nbsp;2, getId()
623 * would always return an empty string, and isSetId() would always return 
624 * <code>false</code> for objects of these classes.
625 <p>
626 * With the addition of 'id' and 'name' attributes on {@link SBase} in Level&nbsp;3
627 * Version&nbsp;2, it became necessary to introduce a new way to interact
628 * with the attributes more consistently in libSBML to avoid breaking
629 * backward compatibility in the behavior of the original 'id' methods.  For
630 * this reason, libSBML provides four functions (getIdAttribute(),
631 * setIdAttribute(String), isSetIdAttribute(), and
632 * unsetIdAttribute()) that always act on the actual 'id' attribute inherited
633 * from {@link SBase}, regardless of the object's type.  <strong>These new methods
634 * should be used instead of the older getId()/setId()/etc. methods</strong>
635 * unless the old behavior is somehow necessary.  Regardless of the Level and
636 * Version of the SBML, these functions allow client applications to use more
637 * generalized code in some situations (for instance, when manipulating
638 * objects that are all known to have identifiers).  If the object in
639 * question does not posess an 'id' attribute according to the SBML
640 * specification for the Level and Version in use, libSBML will not allow the
641 * identifier to be set, nor will it read or write 'id' attributes for those
642 * objects.
643 <p>
644 * <p>
645 * @return integer value indicating success/failure of the
646 * function.   The possible values
647 * returned by this function are:
648 * <ul>
649 * <li> {@link libsbmlConstants#LIBSBML_OPERATION_SUCCESS LIBSBML_OPERATION_SUCCESS}
650 * <li> {@link libsbmlConstants#LIBSBML_OPERATION_FAILED LIBSBML_OPERATION_FAILED}
651 *
652 * </ul> <p>
653 * @see #getIdAttribute()
654 * @see #setIdAttribute(String sid)
655 * @see #isSetIdAttribute()
656 * @see #unsetIdAttribute()
657   */ public
658 int unsetId() {
659    return libsbmlJNI.GraphicalObject_unsetId(swigCPtr, this);
660  }
